# UniHacker 方法 - 需要重新签名授权文件

## 🔍 问题分析

### 当前状态
- ✅ app.asar 中的 PEM 公钥证书已成功替换
- ✅ Hub 配置已修改
- ❌ 授权文件签名验证失败

### 问题原因
授权文件（`Unity_lic.ulf`）中的签名是用**旧的私钥**签名的，但 app.asar 中的公钥已经替换为**新的公钥**。

公钥和私钥不匹配，导致签名验证失败。

## 💡 解决方案

需要用与新公钥证书对应的私钥重新签名授权文件。

### 方案 1：使用 UniHacker 已签名的授权文件

从 UniHacker 项目中获取已经用正确私钥签名的授权文件。

### 方案 2：生成新的密钥对并签名

1. 生成 RSA 密钥对
2. 将公钥替换到 app.asar
3. 用私钥签名授权文件

### 方案 3：找到 MOD_PEM 对应的私钥

UniFree 代码中有 `MOD_PEM_B64` 常量（公钥），需要找到对应的私钥。
